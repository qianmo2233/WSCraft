WSCraft开发手册

这是一款什么垃圾插件？

WSCraft是一款为Bukkti服务器提供WebSocket支持的前置插件

如果你需要一个网站来显示某些实时数据，那么你就需要他

注意：本插件在Bukkit1.16.2测试通过，理论上其他版本也可以，请自行测试

1.检测前置

检测前置十分简单，这里列举两个方法：

  1. 在plugin.yml添加依赖
  2. 在onEnable()方法添加如下检测 ：
public void onEnable() {
  //do something...
  if(getServer().getPluginManager().getPlugin("WSCraft") == null) {
    this.getLogger().warning("未检测到WSCraft，本插件需要WSCraft插件支持！！");
    this.getLogger().warning("正在关闭此插件")
    getServer().getPluginManager().disablePlugin(this);
  }
  //do something...
}

个人认为方法二更好，毕竟可以显示自定义信息嘛(*^▽^*)

2.发送消息

WSCraft提供了两个静态方法：

方法名 返回类型 类型 参数名 描述
sendMsgToOne Boolean String ID, String Msg 发送给某ID
sendMsgToAll Boolean String Msg 群发

直接调用API.sendMsgToOne(someone, message);或API.sendMsgToAll(message);即可

请谨慎使用群发，因为未经鉴权的对方的身份是未知的

注意：所发送内容必须使用JSON字符串格式，否则WSCraft将报错且不发送该信息

3.接收消息

既然能发送消息，总得也要接收消息吧

对于接收消息，WSCraft提供了一个抽象类以供编写

首先，继承WSCraft类，然后对其中的抽象方法进行编写：

public class callBack extends WSCraft {
    public void onMesssage(String ID, String Msg) {
        //当收到消息时触发
    }

    public void onConnect(String ID) {
        //当有新连接加入时触发
    }

    public void onDisconnect(String ID, String Reason) {
        //当有连接断开时触发
    }

    public void onException(String ID, Exception e) {
        //当出现问题时触发
    }
}

然后为这个类注册一个Bukkit监听器即可

这样就好啦~

注意：如果接收到的数据非JSON字符串格式，WSCraft将会报错，且不会触发onMsg方法

3.玩家绑定与鉴权

鬼知道这是谁的连接~

对于上述问题，就需要将ID与玩家(或其他奇奇怪怪的东西)绑定

由于绑定方式因插件实际需求而异，故此操作需要自行完成

这是一个事关安全性的必要操作，请勿跳过此步

如何进行此操作？

可以创建一个HashMap，来储存ID和玩家(名)

在玩家建立连接时，需要发送一段特定的信息进行鉴权

鉴权成功后，储存ID和玩家(名)

发送消息时，从自己建立的HashMap中遍历发送就好了

4.数据特征与加密

对面发来的是什么鬼东西~

WSCraft接收所有发来的JSON字符串，由于每个插件需要的数据不同，就需要对接收到的数据进行过滤

强烈建议对数据进行加密操作，防止被其他插件拦截

数据特征可以像这样：

{
    "Auth-Json": "Auth-fewf7t7vfsfkjb",//这就是特征
    "name": "8gYLgi478VBLK",//加密数据防拦截
    "password": "Af87etfVHJK"//加密数据防拦截
}
